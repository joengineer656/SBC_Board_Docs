This page gives you a high-level orientation of the SBC Board's physical design and the main subsystems you will work with. Understanding the board's layout before you connect peripherals or write your first program helps you avoid common mistakes and makes the detailed reference pages — like the pinout and full specifications — easier to navigate. For electrical specifications and exact signal assignments, refer to the [Specifications](/hardware/specifications) and [Pinout Reference](/hardware/pinout) pages.

## Main Components

SBC Board integrates all the core building blocks of a capable embedded Linux platform onto a single compact PCB. The cards below summarize the key subsystems and what they enable.

<CardGroup cols={2}>
  <Card title="Processor" icon="microchip" href="/hardware/specifications">
    A multi-core ARM application processor running at up to several GHz, with an integrated GPU for hardware-accelerated graphics and media decoding. The SoC also includes dedicated hardware blocks for cryptography, video encoding, and real-time I/O.
  </Card>
  <Card title="Memory & Storage" icon="hard-drive" href="/hardware/specifications">
    Onboard LPDDR RAM soldered directly to the PCB for low-latency access. A microSD card slot provides primary removable storage. Select variants include an eMMC module or an M.2 slot for higher-speed solid-state storage.
  </Card>
  <Card title="Connectivity" icon="wifi" href="/connectivity/networking">
    Integrated dual-band Wi-Fi and Bluetooth via a certified wireless module, plus a full-speed Gigabit Ethernet port. USB ports provide both host and device connectivity, and a USB-C port handles power delivery.
  </Card>
  <Card title="GPIO Header" icon="diagram-project" href="/hardware/pinout">
    A 40-pin through-hole header compatible with a wide ecosystem of HATs, add-on boards, and breadboard wiring. Pins expose UART, SPI, I2C, PWM, and general-purpose digital I/O at 3.3 V logic levels.
  </Card>
  <Card title="Power" icon="bolt" href="/hardware/specifications">
    Powered via a USB-C port requiring 5 V / 3 A (15 W). An onboard power management IC regulates supply rails for the SoC, memory, and peripherals. A power LED and a user-programmable activity LED provide basic status indication.
  </Card>
  <Card title="Video & Audio" icon="display" href="/hardware/specifications">
    Full-size HDMI output supporting up to 4K resolution at 60 Hz (resolution subject to SoC capability). A 3.5 mm combination audio jack provides analog stereo output and microphone input. HDMI also carries digital audio (HDMI ARC not supported on all revisions).
  </Card>
</CardGroup>

## Board Dimensions

SBC Board follows a form factor similar to the widely adopted credit-card SBC standard, making it compatible with a broad range of off-the-shelf cases and mounting hardware.

| Attribute              | Value                              |
|------------------------|------------------------------------|
| Length                 | 85 mm                              |
| Width                  | 56 mm                              |
| Height (PCB only)      | ~1.6 mm (PCB thickness)            |
| Height (tallest component) | ~15 mm (from PCB top surface)  |
| Mounting holes         | 4 × M2.5, standard corner layout   |
| Mounting hole pitch    | 58 mm × 49 mm (center to center)   |
| Weight (board only)    | ~45 g                              |
| Operating temperature  | 0 °C to 50 °C (non-condensing)     |

<Tip>
  Before attaching any HATs, ribbon cables, or custom peripherals to the GPIO header, review the [Pinout Reference](/hardware/pinout) page. Applying incorrect voltages or connecting signals to the wrong pins can permanently damage the board or the attached accessory. The GPIO pins operate at **3.3 V logic** and are not 5 V tolerant unless explicitly stated.
</Tip>

## What's Supported

SBC Board is designed to work with a range of community-supported and officially maintained operating systems, and to be programmed in whichever language best fits your project.

<Tabs>
  <Tab title="Operating Systems">
    The following Linux distributions are actively tested and supported on SBC Board. Other Debian- and Arch-based distributions may also work but are community-supported only.

    | Distribution              | Support Level     | Desktop Environment      | Notes                                      |
    |---------------------------|-------------------|--------------------------|--------------------------------------------|
    | SBC Board OS (Recommended)| ✅ Official        | Customized XFCE / headless | Purpose-built image with all drivers pre-installed |
    | Raspberry Pi OS (Lite)    | ✅ Official        | None (headless)          | Minimal image; ideal for server workloads  |
    | Ubuntu Server LTS         | ✅ Official        | None (headless)          | Long-term support, strong package ecosystem|
    | Ubuntu Desktop LTS        | ✅ Official        | GNOME                    | Full desktop; requires ≥ 2 GB RAM variant  |
    | Debian Stable (arm64)     | 🔧 Community       | Optional                 | Vanilla Debian; manual driver setup needed |
    | Arch Linux ARM            | 🔧 Community       | Optional                 | Rolling release; for experienced users     |
    | Manjaro ARM               | 🔧 Community       | KDE Plasma / XFCE        | User-friendly Arch derivative              |

    <Note>
      Always download OS images from the [official OS Installation page](/software/os-installation) or from the linked official distribution mirrors. Third-party images may not include the correct device tree or firmware blobs for full hardware support.
    </Note>
  </Tab>
  <Tab title="Programming Languages">
    SBC Board runs a full Linux userspace, so virtually any language with an ARM64 (AArch64) build can be used. The following languages have well-tested libraries for GPIO, I2C, SPI, and other hardware interfaces.

    | Language      | GPIO / Hardware Library          | Installation                        | Best For                                   |
    |---------------|----------------------------------|-------------------------------------|--------------------------------------------|
    | **Python 3**  | `gpiozero`, `RPi.GPIO`, `smbus2` | Pre-installed on SBC Board OS        | Rapid prototyping, scripting, education     |
    | **C / C++**   | `libgpiod`, `wiringPi` (legacy)  | `apt install libgpiod-dev`           | Performance-critical bare-metal control     |
    | **Rust**      | `rppal`, `linux-embedded-hal`    | Install via `rustup`                 | Safe systems programming, IoT firmware      |
    | **Node.js**   | `onoff`, `i2c-bus`, `spi-device` | `apt install nodejs` or via `nvm`    | Event-driven apps, web dashboards, MQTT     |
    | **Go**        | `periph.io`, `go-rpio`           | Download from golang.org/dl          | Concurrent services, lightweight daemons    |

    <Tip>
      Python 3 with `gpiozero` is the fastest way to get hardware blinking or reading sensors. Once you have a working prototype, you can port performance-sensitive loops to C/C++ or Rust without changing your wiring or circuit.
    </Tip>
  </Tab>
</Tabs>
