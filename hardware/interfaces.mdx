The board packs a comprehensive set of interfaces into a compact form factor, giving you access to high-speed USB, video output, camera and display ribbons, wired and wireless networking, and a full 40-pin expansion header. Understanding what each connector does and how it interacts with the operating system helps you plan your project wiring and driver configuration before you start assembly.

## USB

The board provides three USB connections that cover both host and device roles.

**2× USB 3.0 Type-A (Host)**

These ports operate as standard USB hosts at **SuperSpeed (5 Gbps)**. You can connect keyboards, mice, flash drives, USB cameras, SDR receivers, and virtually any USB 3.0 or USB 2.0 peripheral. Each port shares the board's total USB bus bandwidth, so avoid saturating both ports simultaneously with high-throughput devices such as two simultaneous USB 3.0 SSDs.

| Attribute      | Value                  |
|----------------|------------------------|
| Connector      | USB Type-A (×2)        |
| Standard       | USB 3.0 (SuperSpeed)   |
| Max Throughput | 5 Gbps per bus         |
| Current Supply | Up to 900 mA per port  |

**1× USB-C (OTG + Power Delivery)**

The USB-C port serves a dual role: it is the primary **power input** for the board (5 V / 3 A), and it also supports **USB On-The-Go (OTG)** data, allowing the board to operate as a USB device (e.g., a USB gadget, serial console, or network interface) when connected to a host computer.

<Note>
  The USB-C port handles both power delivery and OTG data simultaneously, but it cannot act as a USB 3.0 host port — that role belongs exclusively to the two Type-A ports. When you use USB-C for OTG data, ensure your power supply also supports USB-C Power Delivery or that you have an alternative power source connected.
</Note>

| Attribute   | Value                          |
|-------------|--------------------------------|
| Connector   | USB Type-C                     |
| Standard    | USB 2.0 OTG + Power Delivery   |
| Power Input | 5 V DC, up to 3 A (15 W)       |
| OTG Modes   | Device, Host (limited)         |

**OS Driver Support:** All three USB ports are natively supported by the Linux kernel with no additional driver installation required.

## HDMI

The full-size HDMI 2.0 connector delivers digital video and audio to monitors, televisions, and capture cards.

| Attribute      | Value                                   |
|----------------|-----------------------------------------|
| Connector      | HDMI Type-A (full-size)                 |
| Standard       | HDMI 2.0                                |
| Max Resolution | 4K (3840 × 2160) @ 30 fps              |
| 1080p          | 1920 × 1080 @ 60 fps                   |
| Audio          | HDMI audio passthrough (stereo and multichannel) |
| CEC            | Supported                               |

The HDMI output is driven directly by the SoC's display engine. You can output 4K video at 30 fps or full 1080p at 60 fps. Use a certified **High-Speed HDMI cable** (18 Gbps rated) for 4K operation — lower-grade cables may cause flickering or fallback to a lower resolution.

**OS Driver Support:** The HDMI output is initialized by the bootloader and handed off to the kernel DRM/KMS subsystem. Standard Linux desktop environments detect the connected display automatically via EDID negotiation.

## Camera (CSI)

The MIPI CSI-2 connector lets you attach a camera module directly to the SoC's image signal processor, bypassing USB entirely for lower latency and higher throughput.

| Attribute      | Value                                  |
|----------------|----------------------------------------|
| Connector      | 15-pin FPC ribbon (CSI-2)              |
| Lanes          | 2-lane MIPI CSI-2                      |
| Max Resolution | 12 MP sensor support                   |
| Typical Modules | 8 MP (1080p60 / 4K15), 12 MP (4K30)  |
| Interface      | Direct to SoC ISP                      |

To attach a camera module, gently lift the locking tab on the CSI connector, insert the ribbon cable with the contacts facing toward the board, and press the tab back down until it clicks. The cable is fragile — avoid flexing it sharply.

**OS Driver Support:** Camera modules require a device tree overlay to be enabled in the boot configuration. After enabling the correct overlay, the camera appears as a V4L2 device (`/dev/video0`) and is compatible with `libcamera`, `GStreamer`, and `ffmpeg`.

## Display (DSI)

The MIPI DSI connector provides a direct link to LCD and OLED panel displays, most commonly the official single-board computer touchscreen accessory.

| Attribute  | Value                              |
|------------|------------------------------------|
| Connector  | 15-pin FPC ribbon (DSI)            |
| Lanes      | 2-lane MIPI DSI                    |
| Resolution | Up to 1280 × 800 typical           |
| Touch      | Supported via I2C (panel-dependent)|

Connect the DSI display ribbon with the contacts facing away from the board (toward the USB ports side). Power the display panel using the dedicated display power connector on the panel PCB if your display model requires it.

**OS Driver Support:** The DSI display is enabled via a device tree overlay. Once enabled, it appears as a secondary framebuffer device and can be used alongside or instead of the HDMI output.

## Audio

The board provides both analog and digital audio paths.

**3.5 mm Combo Jack**

The 3.5 mm TRRS connector supports stereo headphone output and mono microphone input simultaneously. Use a 4-pole TRRS cable (CTIA wiring standard) to access both headphone and mic channels with a single connector.

| Attribute  | Value                          |
|------------|--------------------------------|
| Connector  | 3.5 mm TRRS (4-pole)           |
| Output     | Stereo headphone, line-level   |
| Input      | Mono microphone                |
| Standard   | CTIA (tip-ring-ring-sleeve)    |

**I2S Digital Audio**

Four GPIO pins on the 40-pin header (GPIO 18, 19, 20, 21) are shared with the I2S audio interface. This allows you to connect digital audio HATs, DAC boards, and amplifier modules that communicate over the I2S bus for audiophile-quality output without the noise floor of an analog signal path.

| Signal      | GPIO # | Physical Pin |
|-------------|-------:|-------------:|
| I2S BCLK    | GPIO 18 | 12          |
| I2S LRCLK   | GPIO 19 | 35          |
| I2S DATA IN | GPIO 20 | 38          |
| I2S DATA OUT| GPIO 21 | 40          |

**OS Driver Support:** The 3.5 mm jack and I2S bus both appear as ALSA sound cards under Linux. The onboard codec driver loads automatically. I2S HATs typically require an additional device tree overlay to register the external DAC/codec.
