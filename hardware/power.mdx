Providing clean, stable power is one of the most important steps in getting reliable operation from your board. Voltage dips caused by an inadequate power supply are the single most common source of random reboots, filesystem corruption, and mysterious peripheral failures. This guide walks you through choosing the right supply, understanding the on-board power rails, and configuring software-side power management features.

<Warning>
  Always use a quality **5 V / 3 A (15 W) USB-C power supply** from a reputable manufacturer. Cheap or underpowered adapters that cannot sustain 3 A under load will cause the board to brown out, throttle the CPU, or corrupt data on your storage medium. Do not power the board from a laptop USB port — most deliver far less than the required 3 A.
</Warning>

## Power Requirements

The board is powered exclusively through its USB-C port. There is no barrel-jack power input.

| Parameter         | Value                      |
|-------------------|----------------------------|
| Input Voltage     | 5 V DC                     |
| Connector         | USB Type-C                 |
| Minimum Current   | 2 A (no peripherals attached) |
| Recommended Current | 3 A                      |
| Maximum Power     | 15 W                       |

The 3 A recommendation accounts for worst-case simultaneous load: CPU at full clock, both USB 3.0 ports loaded with high-current devices, active HDMI output, and a GPIO HAT. If your project runs peripherals with high current demands (e.g., motors, heating elements) they should always have their own dedicated power supply — never draw such loads through the board.

<Note>
  The board supports **Power over Ethernet (PoE)** when used with a compatible PoE HAT attached to the 40-pin header and the RJ45 port. A PoE HAT negotiates power from a PoE-enabled switch or injector (802.3af / 802.3at) and converts it to 5 V for the board, eliminating the need for a separate USB-C power supply in network-connected deployments.
</Note>

## Power Rails

The board maintains three internal voltage domains. Two are accessible externally on the 40-pin header; the third is internal only.

| Rail   | Source            | Header Pins        | Max Current (External) | Notes                              |
|--------|-------------------|--------------------|------------------------|------------------------------------|
| 5 V    | USB-C input       | Pin 2, Pin 4       | Limited by PSU capacity | Unregulated; reflects USB-C input voltage exactly |
| 3.3 V  | On-board LDO regulator | Pin 1, Pin 17 | 300 mA total (all header loads) | Regulated; powers GPIO logic and HATs |
| 1.8 V  | Internal regulator| Not exposed        | N/A                    | Used internally by SoC I/O banks   |

Keep the following constraints in mind when designing HATs or attaching peripherals to the header:

- The **5 V rail** on pins 2 and 4 is a direct feed from the USB-C input. Any current you draw from these pins comes out of your power supply's budget, so factor it in when sizing your PSU.
- The **3.3 V rail** on pins 1 and 17 is limited to a combined **300 mA** across all attached loads. If your HAT or peripheral needs more than this, provide a separate regulated 3.3 V supply to that device.
- Never attempt to back-power the board by injecting voltage into the 5 V header pins while the USB-C port is also connected unless you are using dedicated power-ORing circuitry.

## Power Management

### Soft Shutdown

Always shut down the operating system gracefully before removing power. Cutting power without a proper shutdown can leave your filesystem in an inconsistent state and corrupt data on your microSD or eMMC storage.

To perform a safe shutdown from the terminal, run:

```bash
sudo shutdown -h now
```

Wait for all activity LEDs on the board to stop blinking and for the system status LED to go dark before disconnecting the power supply.

### GPIO Power Button

You can wire a physical push button to a GPIO pin to trigger a graceful software shutdown, which is useful in enclosures where terminal access is not practical.

Connect a momentary push button between **GPIO 3 (physical pin 5)** and **ground (physical pin 6)**. GPIO 3 has a built-in pull-up resistor, so pressing the button pulls the line low.

Install and configure a button monitor service:

```bash
# Install the gpio-button-shutdown utility (example for Debian-based systems)
sudo apt install python3-gpiozero

# Create a simple shutdown listener script
cat << 'EOF' > /usr/local/bin/power-button.py
from gpiozero import Button
from subprocess import call

button = Button(3, hold_time=2)
button.when_held = lambda: call(['shutdown', '-h', 'now'])

from signal import pause
pause()
EOF

sudo chmod +x /usr/local/bin/power-button.py
```

Add that script to a systemd service so it starts automatically on boot. A 2-second hold prevents accidental shutdowns from a brief button press.

### CPU Frequency Scaling (Low-Power Mode)

The board's CPU governor controls how aggressively the processor scales its clock speed in response to workload. Switching from the default `ondemand` governor to `powersave` significantly reduces power draw and heat output when running light tasks.

View available governors:

```bash
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_available_governors
```

Set the governor for all cores to `powersave`:

```bash
for cpu in /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor; do
  echo powersave | sudo tee "$cpu"
done
```

Use the `cpufreq-set` utility from the `cpufrequtils` package for a cleaner interface:

```bash
sudo apt install cpufrequtils
sudo cpufreq-set -g powersave -r   # Apply to all cores
cpufreq-info                        # Verify current settings
```

To make the governor setting persist across reboots, add it to `/etc/rc.local` or configure it via a `systemd` drop-in for the `cpufrequtils` service.

<Tip>
  When you connect power-hungry USB devices (external hard drives, USB audio interfaces, USB hubs), use a **powered USB hub** with its own PSU rather than plugging them directly into the board's USB 3.0 Type-A ports. Each host port can supply up to 900 mA, but those milliamps come directly out of your board's 5 V power budget. Offloading peripheral power to an external hub keeps your board stable and your PSU headroom healthy.
</Tip>

## Battery & UPS Options

For applications that must survive unexpected power cuts — outdoor sensors, industrial controllers, home automation hubs — you can add battery backup through HAT-form-factor expansion boards that sit directly on the 40-pin header.

<CardGroup cols={2}>
  <Card title="LiPo / Li-ion Battery HATs" icon="battery-full">
    These HATs accept single-cell or dual-cell lithium batteries and include a charging circuit, fuel gauge IC (accessible over I2C), and automatic switchover. The board continues running from battery when mains power is lost, and recharges the battery when power returns.
  </Card>
  <Card title="Supercapacitor UPS HATs" icon="bolt">
    Supercapacitor-based UPS HATs provide a short holdup window (typically 10–30 seconds) at full board power — just long enough to trigger a clean shutdown via GPIO signal. They recharge in seconds and have virtually unlimited cycle life compared to lithium cells.
  </Card>
  <Card title="PoE + Battery Combined HATs" icon="plug">
    Combined HATs integrate PoE power negotiation with a battery backup cell in a single board, ideal for network-edge deployments where you want both cable-based power delivery and resilience against brief outages.
  </Card>
  <Card title="18650 Battery Expansion Boards" icon="car-battery">
    These expansion boards accommodate standard 18650 cylindrical cells in holders, making it easy to use off-the-shelf rechargeable cells. They typically expose remaining capacity and charge/discharge status over I2C at address `0x36` or `0x62`.
  </Card>
</CardGroup>

When evaluating a battery HAT, confirm that it:
- Provides a **power-loss GPIO signal** so your software can initiate a graceful shutdown before the battery is depleted
- Supports **5 V / 3 A output** — some budget HATs can only supply 2 A, which may cause instability under full CPU load
- Has a **charging current compatible** with your battery cell's C-rating to avoid reducing battery lifespan
