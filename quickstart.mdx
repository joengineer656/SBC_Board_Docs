This guide walks you through everything you need to get your SBC Board running for the first time. By the end you will have a fully booted Linux environment that you can access either through a connected display or remotely over SSH. No prior embedded Linux experience is required — just follow the steps below in order.

<Steps>
  <Step title="Gather What You Need">
    Before you begin, collect all the required hardware. Having everything on hand before you start avoids interruptions mid-setup.

    **Required items:**

    - ✅ **SBC Board** — your main board
    - ✅ **microSD card** — 16 GB or larger, Class 10 or A1-rated recommended
    - ✅ **USB-C power supply** — 5 V, 3 A (15 W) minimum; an underpowered supply causes instability
    - ✅ **USB-C cable** — to connect the power supply to the board
    - ✅ **HDMI monitor and cable** — for the initial setup display output
    - ✅ **USB keyboard and mouse** — for navigating the first-boot wizard

    **Optional but useful:**

    - A second computer (Windows, macOS, or Linux) to flash the microSD card
    - A wired Ethernet cable for a more reliable first-boot network connection
    - A case or heatsink if you plan to run the board under sustained load

    <Info>
      Check your power supply's output rating before proceeding. Many phone chargers are rated at only 5 V / 1 A or 5 V / 2 A, which is not sufficient for reliable operation. Using an underpowered supply is the most common cause of random reboots and SD card corruption.
    </Info>
  </Step>

  <Step title="Download and Flash the OS Image">
    Download the latest official OS image and write it to your microSD card using a flashing tool on your computer.

    **1. Download the image**

    Visit the [OS Installation page](/software/os-installation) to find the latest stable image. Download the `.img.xz` or `.img.gz` file to your computer.

    **2. Flash with Raspberry Pi Imager (recommended)**

    [Raspberry Pi Imager](https://www.raspberrypi.com/software/) is a free, cross-platform tool that handles downloading, verifying, and writing images in one step.

    1. Open Raspberry Pi Imager and click **Choose OS → Use custom**.
    2. Select the `.img.xz` file you downloaded.
    3. Click **Choose Storage** and select your microSD card — double-check you have the right drive.
    4. Click the ⚙️ **gear icon** to pre-configure Wi-Fi credentials, hostname, and SSH access before flashing.
    5. Click **Write** and wait for the verify step to complete.

    **Alternatively, use balenaEtcher**

    [balenaEtcher](https://etcher.balena.io/) offers a similarly simple interface: select image → select drive → Flash.

    <Warning>
      Flashing will erase all existing data on the microSD card. Make sure you have selected the correct drive before confirming. Flashing to the wrong device can cause data loss on your computer.
    </Warning>
  </Step>

  <Step title="Insert the microSD Card and Connect Peripherals">
    With the OS image flashed and verified, prepare the board for its first boot.

    1. **Insert the microSD card** — push it into the microSD slot on the underside of the board until it clicks into place.
    2. **Connect the HDMI cable** — plug one end into the board's HDMI port and the other into your monitor.
    3. **Connect your USB keyboard and mouse** — plug them into the available USB-A ports.
    4. **Connect Ethernet (optional but recommended)** — plug in a network cable for a stable internet connection during first boot.
    5. **Do not connect power yet** — wait until all peripherals are attached.

    <Tip>
      Connect the HDMI cable before powering on. Some monitors and display protocols do not support hot-plug detection, so connecting the display after boot may result in no video output.
    </Tip>
  </Step>

  <Step title="Power On and Complete First Boot">
    Connect the USB-C power supply to power on the board. SBC Board has no physical power button — it boots as soon as power is applied.

    1. **Connect the USB-C power cable** to the board's power port. The status LED should illuminate immediately.
    2. **Watch the boot sequence** on your HDMI monitor. You will see boot log messages followed by the first-boot setup wizard.
    3. **Follow the first-boot wizard** to:
       - Set your locale, timezone, and keyboard layout
       - Create a user account and password
       - Connect to Wi-Fi (if you did not pre-configure it during flashing)
       - Accept or decline optional telemetry
    4. The system will apply any pending updates and then drop you into the desktop environment or a login prompt.

    A typical first boot takes **60–90 seconds**. Subsequent boots are faster once the initial setup is complete.

    <Info>
      If the board does not appear to boot (no LED activity, no display output), re-check that the microSD card is fully seated and that your power supply meets the 5 V / 3 A requirement. See the [FAQ](/resources/faq) for further troubleshooting steps.
    </Info>
  </Step>

  <Step title="Connect via SSH or Use the Desktop">
    Once the board has booted, you can interact with it directly through the desktop or remotely over SSH from another computer on the same network.

    **Using the desktop**

    If you connected a monitor, keyboard, and mouse, you are already at the desktop. Open a terminal emulator to start running commands.

    **Connecting via SSH**

    Find the board's IP address from your router's admin panel, from the first-boot wizard screen, or by running `hostname -I` in a local terminal on the board.

    Then, from your computer, run:

    ```bash
    ssh admin@<board-ip-address>
    ```

    Replace `<board-ip-address>` with the actual IP (for example, `192.168.1.42`). When prompted, enter the password you set during the first-boot wizard. If you pre-configured SSH with a public key during flashing, key-based authentication will be used automatically.

    <Tip>
      Set a static IP address or a DHCP reservation in your router for the board's MAC address. This prevents the IP from changing between reboots, making SSH connections more predictable.
    </Tip>
  </Step>
</Steps>
