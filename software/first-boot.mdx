With a freshly flashed microSD card installed, you are ready to power on your board and complete the initial setup. The first boot takes a little longer than subsequent boots because the system automatically expands the root filesystem to fill the entire card — this is normal and only happens once. Follow the steps below to go from a blank board to a fully configured, network-connected system.

## First Boot Setup

<Steps>
  <Step title="Connect HDMI, Keyboard, and Power">
    Before applying power, connect the following peripherals:

    - **Display**: plug an HDMI cable into the board's full-size HDMI port and connect the other end to a monitor or TV.
    - **Keyboard**: connect a USB keyboard to any available USB-A port.
    - **Optional mouse**: required only if you flashed a desktop image.

    Once peripherals are connected, plug the **USB-C power supply** into the board's USB-C port. The board powers on immediately — there is no power button. A red power LED near the USB-C port confirms the board is receiving power.

    <Note>
      Use a power supply rated for at least **5V 3A (15W)**. Underpowered supplies cause random reboots and filesystem corruption. Charging cables and laptop USB ports are not reliable power sources for this board.
    </Note>
  </Step>

  <Step title="Wait for the Boot Sequence to Complete">
    Watch the display for boot messages. On the very first boot the system performs two automatic tasks before presenting a login prompt:

    1. **Filesystem expansion** — the root partition grows to fill the microSD card (~60 seconds).
    2. **First-run configuration** — SSH host keys are generated and default services are started (~30–60 seconds).

    The entire first-boot process typically takes **1.5–2 minutes**. Subsequent boots complete in under 30 seconds. Do not remove power during this phase.

    <Info>
      If the display shows only a rainbow splash screen for more than 30 seconds without any boot text, the image may not have flashed correctly. Re-flash the card and verify the checksum before trying again.
    </Info>
  </Step>

  <Step title="Log In with Default Credentials">
    When the login prompt appears, enter the default credentials:

    | Field | Value |
    |---|---|
    | Username | `admin` |
    | Password | `admin` |

    <Warning>
      The default password is publicly known. **Change it immediately** in the next step before connecting the board to any network. Anyone on the same network can log in with these credentials until you change them.
    </Warning>
  </Step>

  <Step title="Change Your Password">
    Run `passwd` to set a strong, unique password for your account:

    ```bash
    passwd
    ```

    You will be prompted to enter your current password (`admin`), then your new password twice. Choose a password that is at least 12 characters long and not used on any other service.

    ```
    Changing password for admin.
    Current password:
    New password:
    Retype new password:
    passwd: password updated successfully
    ```
  </Step>

  <Step title="Update the System">
    Refresh the package index and apply any updates that have been released since the image was built:

    ```bash
    sudo apt update && sudo apt upgrade -y
    ```

    This step can take several minutes on first run. A full upgrade ensures you have the latest security patches and bug fixes before installing additional software.

    <Tip>
      If `apt upgrade` prompts you about a modified configuration file, pressing **Enter** to keep the currently-installed version is safe for most files unless you know you need the package maintainer's newer version.
    </Tip>
  </Step>
</Steps>

## Enable SSH

Enabling SSH lets you connect to the board remotely from your laptop or desktop, so you no longer need a physical keyboard and monitor attached. Enable and start the SSH service with:

```bash
sudo systemctl enable ssh
sudo systemctl start ssh
```

Confirm the service is running:

```bash
sudo systemctl status ssh
```

The output should show `Active: active (running)`. SSH listens on port **22** by default.

<Note>
  If you pre-configured SSH using Raspberry Pi Imager's settings panel during flashing, the SSH service is already enabled and these commands are not necessary.
</Note>

## Find Your Board's IP Address

Before connecting over SSH, you need to know the board's IP address on your local network. Run either of these commands directly on the board:

```bash
# Show all network interfaces and their IP addresses
ip addr show

# Print only the IP addresses assigned to this host
hostname -I
```

The output of `hostname -I` looks something like:

```
192.168.1.42 fd00::1a2b:3c4d:5e6f
```

The first address (typically `192.168.1.x` or `10.0.0.x`) is your IPv4 address. Use this address to SSH in from another machine:

```bash
ssh admin@192.168.1.42
```

## Configure Wi-Fi

If you are connecting the board to a wireless network, use `nmcli` (NetworkManager command-line interface) to scan for and join a network:

```bash
# List available Wi-Fi networks
sudo nmcli dev wifi list

# Connect to a network
sudo nmcli dev wifi connect "YourSSID" password "YourPassword"
```

Replace `YourSSID` with your network name and `YourPassword` with your Wi-Fi password. NetworkManager saves the connection and reconnects automatically on every boot.

Confirm the connection:

```bash
nmcli connection show --active
```

<Tip>
  If your SSID contains spaces, wrap it in quotes exactly as shown above. NetworkManager handles special characters correctly as long as the SSID and password are quoted.
</Tip>

## Set Timezone and Locale

By default the board ships with the UTC timezone and a minimal locale. Configure these to match your location so that log timestamps, cron jobs, and date-aware applications behave correctly.

**Set your timezone:**

```bash
# List available timezones
timedatectl list-timezones | grep America

# Set your timezone (replace with your own)
sudo timedatectl set-timezone America/New_York
```

**Verify the change:**

```bash
timedatectl
```

**Set your locale:**

```bash
sudo localectl set-locale LANG=en_US.UTF-8
```

Apply the locale change to your current shell session without logging out:

```bash
source /etc/default/locale
```

<Info>
  You can find your region's timezone string by running `timedatectl list-timezones` and piping through `grep` with your country or city name (e.g., `grep Europe`, `grep Asia`).
</Info>
