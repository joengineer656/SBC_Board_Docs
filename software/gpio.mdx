The 40-pin GPIO (General-Purpose Input/Output) header on your SBC gives you direct electrical access to the board's processor, letting you control LEDs, read buttons, drive motors, and communicate with sensors and displays. Each pin can be configured in software as a digital input, digital output, or one of several hardware-function modes (SPI, I2C, UART, PWM). Linux exposes GPIO through two interfaces: the legacy `sysfs` interface (deprecated but still widely documented) and the modern `libgpiod` interface (recommended for new projects). The higher-level language libraries covered in this guide build on top of these interfaces and are the easiest way to get started.

## GPIO Libraries

Choose the library that matches your preferred language and workflow. All examples in this section toggle a single LED connected to **GPIO17** (physical pin 11) as a concrete starting point.

<Tabs>
  <Tab title="Python (gpiozero)">
    `gpiozero` is the recommended Python library for beginners and most projects. It provides a clean, object-oriented API with sensible defaults and built-in support for dozens of common components (LEDs, buttons, motors, sensors).

    **Install:**

    ```bash
    sudo apt install python3-gpiozero
    # or inside a virtual environment:
    pip install gpiozero
    ```

    **Blink an LED:**

    ```python
    from gpiozero import LED
    from time import sleep

    led = LED(17)  # GPIO17, physical pin 11

    while True:
        led.on()
        sleep(1)
        led.off()
        sleep(1)
    ```

    Save this as `blink.py` and run it with:

    ```bash
    python3 blink.py
    ```

    Press **Ctrl+C** to stop. `gpiozero` automatically cleans up pin state when the script exits.

    **Blink using the built-in `blink()` method:**

    ```python
    from gpiozero import LED

    led = LED(17)
    led.blink(on_time=0.5, off_time=0.5)  # blink every 500 ms

    input("Press Enter to stop\n")
    ```
  </Tab>

  <Tab title="Python (RPi.GPIO)">
    `RPi.GPIO` is a lower-level library that gives you direct control over pin mode and state. It requires explicit setup and cleanup calls and uses either BCM or BOARD pin numbering (see [GPIO Numbering](#gpio-numbering) below).

    **Install:**

    ```bash
    sudo apt install python3-rpi.gpio
    # or inside a virtual environment:
    pip install RPi.GPIO
    ```

    **Blink an LED:**

    ```python
    import RPi.GPIO as GPIO
    from time import sleep

    # Use BCM (Broadcom) GPIO numbering
    GPIO.setmode(GPIO.BCM)

    # Set GPIO17 as an output pin
    GPIO.setup(17, GPIO.OUT)

    try:
        while True:
            GPIO.output(17, GPIO.HIGH)  # LED on
            sleep(1)
            GPIO.output(17, GPIO.LOW)   # LED off
            sleep(1)
    except KeyboardInterrupt:
        pass
    finally:
        # Always clean up to reset pin state
        GPIO.cleanup()
    ```

    Save as `blink_rpi.py` and run:

    ```bash
    python3 blink_rpi.py
    ```

    <Note>
      Always call `GPIO.cleanup()` when your script exits — the `try/finally` block above guarantees this even if an exception occurs. Failing to clean up can leave pins in unexpected states for subsequent programs.
    </Note>
  </Tab>

  <Tab title="C (libgpiod)">
    `libgpiod` is the modern, kernel-supported C library for GPIO access. It replaces the old `sysfs` interface and is suitable for performance-sensitive or bare-metal-adjacent applications.

    **Install:**

    ```bash
    sudo apt install libgpiod-dev gpiod
    ```

    **Blink an LED (C source):**

    ```c
    #include <gpiod.h>
    #include <stdio.h>
    #include <unistd.h>

    int main() {
        struct gpiod_chip *chip = gpiod_chip_open("/dev/gpiochip0");
        if (!chip) {
            perror("gpiod_chip_open");
            return 1;
        }

        struct gpiod_line *line = gpiod_chip_get_line(chip, 17);
        if (!line) {
            perror("gpiod_chip_get_line");
            gpiod_chip_close(chip);
            return 1;
        }

        if (gpiod_line_request_output(line, "blink", 0) < 0) {
            perror("gpiod_line_request_output");
            gpiod_chip_close(chip);
            return 1;
        }

        for (int i = 0; i < 10; i++) {
            gpiod_line_set_value(line, 1);  // LED on
            sleep(1);
            gpiod_line_set_value(line, 0);  // LED off
            sleep(1);
        }

        gpiod_line_release(line);
        gpiod_chip_close(chip);
        return 0;
    }
    ```

    **Compile and run:**

    ```bash
    gcc blink.c -o blink -lgpiod
    ./blink
    ```

    **Command-line tools (no code required):**

    The `gpiod` package also installs handy shell utilities:

    ```bash
    # List all GPIO chips and lines
    gpiodetect
    gpioinfo gpiochip0

    # Set a pin high for 5 seconds then release
    gpioset --mode=time -s 5 gpiochip0 17=1

    # Read the current value of a pin
    gpioget gpiochip0 17
    ```
  </Tab>

  <Tab title="Node.js (onoff)">
    The `onoff` package provides a straightforward GPIO API for Node.js, built on top of the Linux sysfs interface.

    **Install:**

    ```bash
    # Ensure Node.js is installed (see /software/package-management)
    npm install onoff
    ```

    **Blink an LED:**

    ```javascript
    const { Gpio } = require('onoff');

    // Export GPIO17 as an output, initially low
    const led = new Gpio(17, 'out');

    let state = 0;

    // Toggle every 1000 ms
    const interval = setInterval(() => {
      state = state === 0 ? 1 : 0;
      led.writeSync(state);
      console.log(`LED ${state === 1 ? 'ON' : 'OFF'}`);
    }, 1000);

    // Clean up after 10 seconds
    setTimeout(() => {
      clearInterval(interval);
      led.writeSync(0);   // turn off LED
      led.unexport();     // release the GPIO resource
      console.log('Done');
    }, 10000);
    ```

    Save as `blink.js` and run:

    ```bash
    node blink.js
    ```

    **Read a pin asynchronously:**

    ```javascript
    const { Gpio } = require('onoff');

    const button = new Gpio(27, 'in', 'both', { debounceTimeout: 10 });

    button.watch((err, value) => {
      if (err) throw err;
      console.log(`Button state changed: ${value}`);
    });

    // Clean up on Ctrl+C
    process.on('SIGINT', () => {
      button.unexport();
    });
    ```
  </Tab>
</Tabs>

## GPIO Numbering

Two pin numbering schemes are in common use, and confusing them is one of the most frequent sources of bugs in GPIO projects.

| Scheme | Also Called | Description |
|---|---|---|
| **BCM** | GPIO numbering | Refers to the Broadcom SoC pin number (e.g., GPIO17, GPIO27). Used by `gpiozero`, `libgpiod`, and is the default in this documentation. |
| **BOARD** | Physical numbering | Refers to the physical pin position on the 40-pin header (1–40, counting left-to-right, top-to-bottom with the header at the top). |

For example, **GPIO17** in BCM numbering corresponds to **physical pin 11** on the header.

Always specify which scheme your code uses:

```python
# RPi.GPIO — set the numbering mode explicitly at the top of every script
GPIO.setmode(GPIO.BCM)   # use BCM (GPIO17, GPIO27, ...)
# or
GPIO.setmode(GPIO.BOARD) # use physical pin numbers (11, 13, ...)
```

`gpiozero` always uses BCM numbering. `libgpiod` uses the kernel line offset, which maps directly to BCM numbers on this board.

<Tip>
  Print the pinout diagram and tape it near your workbench. You can also display a pin map in the terminal by running `pinout` (requires the `python3-gpiozero` package).
</Tip>

See the [Hardware Pinout Reference](/hardware/pinout) for the complete pin map with BCM numbers, physical positions, and alternate functions (I2C, SPI, UART, PWM) labeled for every pin.

## Reading a Button

Buttons and switches are digital inputs. Use a pull-up resistor to hold the pin HIGH when the button is open, and pull it LOW when the button is pressed. The SBC's GPIO pins have software-configurable internal pull-up and pull-down resistors, so no external resistor is required in most cases.

Wire your button between **GPIO27 (physical pin 13)** and **GND (physical pin 14)**.

```python
from gpiozero import Button
from time import sleep

# Button on GPIO27 with internal pull-up resistor enabled
# Pin reads HIGH (1) when open, LOW (0) when pressed
button = Button(27, pull_up=True)

print("Press the button (Ctrl+C to exit)...")

while True:
    if button.is_pressed:
        print("Button pressed!")
    sleep(0.05)  # poll every 50 ms
```

**Using event callbacks instead of polling:**

```python
from gpiozero import Button
from signal import pause

button = Button(27, pull_up=True)

def on_press():
    print("Button pressed!")

def on_release():
    print("Button released!")

button.when_pressed  = on_press
button.when_released = on_release

# Block the main thread and wait for events
pause()
```

The callback approach is more efficient than polling because it does not consume CPU while waiting for a press.

## PWM Output

Pulse-Width Modulation (PWM) lets you vary the effective voltage on a pin by switching it on and off very rapidly. This is commonly used to control LED brightness, servo position, and motor speed.

### Controlling LED Brightness

Wire an LED between **GPIO18 (physical pin 12)** — one of the hardware PWM-capable pins — and GND through a 330Ω resistor.

```python
from gpiozero import PWMLED
from time import sleep

# PWMLED accepts a value from 0.0 (off) to 1.0 (full brightness)
led = PWMLED(18)

# Fade in from off to full brightness
print("Fading in...")
for brightness in range(0, 101, 5):
    led.value = brightness / 100
    sleep(0.05)

# Fade out from full brightness to off
print("Fading out...")
for brightness in range(100, -1, -5):
    led.value = brightness / 100
    sleep(0.05)

led.off()
```

**Smooth continuous pulse using the built-in method:**

```python
from gpiozero import PWMLED
from signal import pause

led = PWMLED(18)

# Smoothly pulse the LED — fade in and out repeatedly
led.pulse(fade_in_time=1, fade_out_time=1)

pause()
```

### Hardware PWM with RPi.GPIO

```python
import RPi.GPIO as GPIO
from time import sleep

GPIO.setmode(GPIO.BCM)
GPIO.setup(18, GPIO.OUT)

# Create a PWM object: pin 18, frequency 1000 Hz
pwm = GPIO.PWM(18, 1000)
pwm.start(0)  # start with 0% duty cycle (off)

try:
    # Sweep duty cycle from 0% to 100% and back
    while True:
        for duty in range(0, 101, 5):
            pwm.ChangeDutyCycle(duty)
            sleep(0.05)
        for duty in range(100, -1, -5):
            pwm.ChangeDutyCycle(duty)
            sleep(0.05)
except KeyboardInterrupt:
    pass
finally:
    pwm.stop()
    GPIO.cleanup()
```

<Tip>
  Prefer **GPIO18** (physical pin 12) or **GPIO19** (physical pin 35) for PWM — these are connected to the hardware PWM peripheral and produce a clean, CPU-independent signal. Software PWM on any other GPIO pin works but can stutter under CPU load.
</Tip>

---

<Warning>
  **Never exceed the GPIO electrical limits.** The GPIO pins on this board operate at **3.3V logic** — connecting a 5V signal directly to any GPIO pin will permanently damage the SoC. When interfacing with 5V devices, use a level shifter. Additionally, each GPIO pin can safely source or sink a maximum of **16mA**, with a total budget of 50mA across all pins simultaneously. Always use a current-limiting resistor (typically 330Ω–1kΩ) in series with LEDs, and never attempt to drive motors, relays, or high-current loads directly from a GPIO pin — use a transistor, MOSFET, or dedicated driver IC instead.
</Warning>
