Before your board can run any software, it needs an operating system flashed onto a microSD card. This guide walks you through selecting a supported OS, verifying the integrity of your downloaded image, and writing it to storage — whether you prefer a graphical tool or the command line.

## Supported Operating Systems

<Tabs>
  <Tab title="Linux">
    Linux is the primary and best-supported platform for this board. All peripherals, GPIO interfaces, and hardware accelerators have tested drivers on the following distributions:

    | Distribution | Version | Image Type | Notes |
    |---|---|---|---|
    | **Ubuntu** | 22.04 LTS (Jammy) | Server / Desktop | Recommended for most users |
    | **Debian** | 12 (Bookworm) | Minimal / Standard | Lightweight, long-term stable |
    | **Raspberry Pi OS** | Latest (Bookworm-based) | Lite / Full | Familiar tooling, large community |

    <Note>
      Ubuntu 22.04 LTS is the primary tested distribution. Examples throughout this documentation use Ubuntu unless otherwise noted.
    </Note>
  </Tab>
  <Tab title="Other">
    The following systems are available for advanced users and are considered **experimental**. Expect limited hardware support and reduced driver compatibility.

    | Distribution | Status | Notes |
    |---|---|---|
    | **FreeBSD 14** | Experimental | Basic boot confirmed; GPIO and Wi-Fi drivers incomplete |

    <Warning>
      Experimental images are not covered by standard support. Community forums are the best resource for troubleshooting non-Linux installations.
    </Warning>
  </Tab>
</Tabs>

## What You Need

Gather the following before you begin:

<CardGroup cols={3}>
  <Card title="microSD Card" icon="sd-card">
    Minimum **16 GB**, Class 10 or UHS-I speed rating. A 32 GB card is recommended for desktop images with room to grow.
  </Card>
  <Card title="Card Reader" icon="usb">
    A USB or built-in SD card reader connected to your computer. Avoid cheap adapters — slow write speeds can produce corrupted images.
  </Card>
  <Card title="Imaging Software" icon="computer">
    **Raspberry Pi Imager** (recommended for beginners) or the `dd` command-line tool available on Linux and macOS.
  </Card>
</CardGroup>

## Flashing the OS Image

<Steps>
  <Step title="Download the OS Image">
    Navigate to the official image download page and grab the compressed image for your chosen OS. Images use the `.img.xz` format (xz-compressed raw disk image).

    ```bash
    # Example: download Ubuntu 22.04 LTS for the SBC
    wget https://downloads.example-sbc.io/images/ubuntu-22.04-sbc-arm64.img.xz
    ```

    <Tip>
      Download images over a wired connection when possible. A partial download produces a corrupt image that will fail the checksum step below.
    </Tip>
  </Step>

  <Step title="Verify the Image Checksum">
    Every image ships with a corresponding `SHA256SUMS` file. Verifying the checksum confirms the file downloaded completely and has not been tampered with.

    ```bash
    # Download the checksum file
    wget https://downloads.example-sbc.io/images/ubuntu-22.04-sbc-arm64.img.xz.sha256

    # Verify — output should read "OK"
    sha256sum -c ubuntu-22.04-sbc-arm64.img.xz.sha256
    ```

    You can also verify manually by comparing hashes:

    ```bash
    sha256sum ubuntu-22.04-sbc-arm64.img.xz
    # Compare the output hash against the value published on the download page
    ```

    If the hash does not match, delete the file and re-download it before continuing.
  </Step>

  <Step title="Flash the Image to Your microSD Card">
    Choose your preferred flashing method below. Both produce identical results.

    <Tip>
      **New to this process?** Use Raspberry Pi Imager — it handles decompression automatically and makes selecting the correct destination disk much harder to get wrong.
    </Tip>

    <Tabs>
      <Tab title="Raspberry Pi Imager (GUI)">
        1. Download and install [Raspberry Pi Imager](https://www.raspberrypi.com/software/) for your OS.
        2. Open the application and click **Choose OS → Use custom**.
        3. Select the downloaded `.img.xz` file — Imager decompresses it automatically.
        4. Click **Choose Storage** and select your microSD card.
        5. Click the **settings gear** icon to optionally pre-configure hostname, SSH, Wi-Fi, and user credentials.
        6. Click **Write** and confirm.

        Imager will flash and verify the card automatically. This typically takes 3–8 minutes depending on card speed.
      </Tab>
      <Tab title="dd (Command Line)">
        First, identify the correct device node for your microSD card:

        ```bash
        # Linux: list block devices before and after inserting the card
        lsblk

        # macOS: list disks
        diskutil list
        ```

        Decompress and flash in a single pipeline:

        ```bash
        # Decompress then flash (Linux/macOS)
        xz -dc ubuntu-22.04-sbc-arm64.img.xz | sudo dd of=/dev/sdX bs=4M status=progress

        # Or: decompress first, then flash
        xz -d ubuntu-22.04-sbc-arm64.img.xz
        sudo dd if=ubuntu-22.04-sbc-arm64.img of=/dev/sdX bs=4M status=progress && sync
        ```

        Replace `/dev/sdX` with your actual device node (e.g., `/dev/sdb`, `/dev/mmcblk0`).

        <Warning>
          **Double-check the `of=` device path before pressing Enter.** The `dd` command writes directly to the device you specify with no confirmation prompt. Pointing it at the wrong disk — including your system drive — will **permanently destroy all data** on that device. Run `lsblk` again immediately before flashing to confirm the correct path.
        </Warning>

        Wait for the `sync` command to complete before removing the card. The `status=progress` flag shows write speed and progress.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Eject the Card and Insert It into Your Board">
    Safely eject the microSD card from your computer to flush any remaining write buffers:

    ```bash
    # Linux
    sudo eject /dev/sdX

    # macOS
    diskutil eject /dev/diskX
    ```

    Once ejected, insert the microSD card into the card slot on the **underside** of the board with the gold contacts facing the board. The card clicks into place when fully seated.

    You are now ready to proceed to [First Boot and Initial Configuration](/software/first-boot).
  </Step>
</Steps>

## Troubleshooting

<Accordion title="The board does not boot after flashing">
  Verify that the microSD card is fully seated in the slot — it should click in. Re-flash the card using a fresh download and confirm the checksum passes. Try a different card if the problem persists; some off-brand cards have write errors that produce a silent but corrupt flash.
</Accordion>

<Accordion title="dd reports 'permission denied'">
  You must run `dd` with `sudo`. Also confirm no partition on the card is currently mounted — unmount any auto-mounted partitions with `sudo umount /dev/sdX1` (and any other partitions) before flashing.
</Accordion>

<Accordion title="sha256sum reports a mismatch">
  The download is corrupt or incomplete. Delete the file, clear any partial downloads, and re-download. If the mismatch persists, check the official download page to confirm the published checksum matches what you are comparing against.
</Accordion>
