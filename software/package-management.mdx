Your SBC runs a standard Debian-based Linux distribution, which means the full ecosystem of Linux software is available to you. This guide covers the tools you will use most often: the APT package manager for system software, `pip` and virtual environments for Python, NodeSource for Node.js, snap for sandboxed applications, and compiling from source for software that is not yet packaged. Keeping your system updated and knowing how to find, install, and remove packages cleanly will save you a great deal of time as your projects grow.

## APT Package Manager

APT (Advanced Package Tool) is the primary way to install software on Debian and Ubuntu systems. It fetches packages from configured repositories, resolves dependencies automatically, and handles upgrades and removals.

Here are the commands you will use most often:

```bash
# Refresh the local package index from all configured repositories
sudo apt update

# Upgrade all installed packages to their latest available versions
sudo apt upgrade

# Install a specific package (replace <package> with the package name)
sudo apt install <package>

# Search for a package by name or keyword
apt search <keyword>

# Show details about a package before installing
apt show <package>

# Remove a package but keep its configuration files
sudo apt remove <package>

# Remove a package and delete its configuration files
sudo apt purge <package>

# Remove packages that were installed as dependencies but are no longer needed
sudo apt autoremove

# Clear the local cache of downloaded package files
sudo apt clean
```

<Tip>
  Run `sudo apt update` before every `apt install` session to make sure you are installing the latest available version of a package. Skipping the update step can result in dependency conflicts when the local index is stale.
</Tip>

### Practical Examples

```bash
# Install common development tools in one command
sudo apt install -y build-essential git curl wget vim

# Install hardware interface libraries
sudo apt install -y i2c-tools spi-tools

# Check whether a package is already installed
dpkg -l | grep <package>
```

## Installing Python Libraries

Python 3 is pre-installed on all supported images. Use `pip` to install third-party libraries, and use **virtual environments** to keep project dependencies isolated from the system Python installation.

```bash
# Install pip and the venv module if not already present
sudo apt install python3-pip python3-venv

# Create a virtual environment in your home directory
python3 -m venv ~/myenv

# Activate the virtual environment
source ~/myenv/bin/activate

# Your prompt changes to show the active environment:
# (myenv) admin@sbc:~$

# Install GPIO and hardware libraries inside the environment
pip install RPi.GPIO gpiozero

# Install additional popular libraries
pip install adafruit-blinka smbus2 spidev

# List installed packages in the current environment
pip list

# Deactivate the virtual environment when done
deactivate
```

<Info>
  Always activate a virtual environment before running `pip install`. Installing packages with `sudo pip install` at the system level can conflict with APT-managed Python packages and break system tools that depend on specific library versions.
</Info>

### Managing Requirements Files

Track your project's dependencies so others (or a fresh install) can reproduce them:

```bash
# Save all current environment dependencies to a file
pip freeze > requirements.txt

# Restore all dependencies from the file on a new system or environment
pip install -r requirements.txt
```

## Installing Node.js

The version of Node.js in the default APT repositories is often outdated. Use the **NodeSource** distribution to install a current LTS release:

```bash
# Add the NodeSource repository for Node.js 20.x LTS
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# Install Node.js (includes npm)
sudo apt install -y nodejs

# Verify the installation
node --version
npm --version
```

<Note>
  The `curl | bash` pattern downloads and executes a shell script with elevated privileges. This is the officially supported NodeSource installation method. Review the script at `https://deb.nodesource.com/setup_20.x` if you want to inspect it before running.
</Note>

### Using npm

```bash
# Install a package globally (available system-wide)
sudo npm install -g <package>

# Install packages locally for a project
cd ~/my-project
npm init -y
npm install onoff johnny-five

# Update all global packages
sudo npm update -g
```

## Snap Packages

Snap is a sandboxed packaging format maintained by Canonical. It lets you install self-contained applications that bundle their own dependencies, making it easier to run software that is not available in the standard APT repositories or requires a specific version.

```bash
# Ensure snapd is installed and running
sudo apt install snapd
sudo systemctl enable --now snapd

# Install a snap package (example: the micro text editor)
sudo snap install micro --classic

# Install a specific channel/version
sudo snap install code --classic

# List installed snaps
snap list

# Update all installed snaps
sudo snap refresh

# Remove a snap package
sudo snap remove <package>
```

<Info>
  The `--classic` flag grants the snap full system access, similar to a traditionally installed application. Most developer tools require this flag. Snaps without `--classic` run in strict confinement and can only access their own data directory.
</Info>

## Building from Source

Some libraries — especially hardware-specific ones — are not yet available as pre-built packages. Building from source gives you access to the latest version and lets you apply custom patches or compile with specific options.

```bash
# Install the essential build toolchain
sudo apt install -y build-essential cmake git

# Clone the source repository
git clone https://github.com/example/library.git
cd library

# Standard autotools build process
./configure
make
sudo make install

# CMake-based build process (common for modern C/C++ projects)
mkdir build && cd build
cmake ..
make -j$(nproc)
sudo make install

# Refresh the dynamic linker cache after installing new libraries
sudo ldconfig
```

<Tip>
  The `-j$(nproc)` flag tells `make` to use all available CPU cores in parallel, significantly speeding up compilation. On a quad-core board, this can reduce a 10-minute build to under 3 minutes.
</Tip>

### Uninstalling Source-Built Software

Keep a record of where files were installed (`sudo make install` often outputs a list) so you can remove them manually if needed. If the project supports it, run `sudo make uninstall` from the same build directory.

## Keeping Your System Updated

<Tip>
  Run system updates regularly — at minimum once a week on boards connected to the internet. Security vulnerabilities in the Linux kernel, SSH daemon, and system libraries are patched frequently, and staying current is the single most effective way to keep your board secure.
</Tip>

Automate unattended security upgrades to stay protected without manual intervention:

```bash
# Install the unattended-upgrades package
sudo apt install unattended-upgrades

# Enable automatic security updates
sudo dpkg-reconfigure --priority=low unattended-upgrades
```

For a single-command full system refresh, combine `update`, `upgrade`, and cleanup:

```bash
sudo apt update && sudo apt upgrade -y && sudo apt autoremove -y && sudo apt clean
```
